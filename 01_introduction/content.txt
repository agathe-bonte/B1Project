Introduction & Bases du Web

Comprendre le rôle d’un développeur front-end

Le développeur front-end est le métier qui englobe tout ce qui est lié à l’interface utilisateur.

Ergonomie: est la discipline qui étudie les interactions entre l’être humain et les systèmes afin d’optimiser le confort et l’efficacité des personnes dans l’utilisation du site. 
	•	L’ergonomie visuelle : fidélité des maquettes, harmonie, interaction, animations… 
	•	L’ergonomie fonctionnelle du site - navigation au clavier, actions tactiles… 

Compatibilité : désigne la capacité d’un site ou d’une application à fonctionner  correctement sur différents environnement. 
	•	Appareils (ordinateurs, smartphones, tablettes, TV connectées),
	•	Navigateurs web (Chrome, Firefox, Safari, Edge, etc.)

Accessibilité : consiste à concevoir des sites et applications de manière à ce qu’ils soient utilisables pour tout le monde notamment aux personnes en situation de handicap. Cela repose sur des normes comme les WCAG (Web Content Accessibility Guidelines), qui fixent des principes.
	•	Un site web accessible propose des boutons suffisamment grands, bien contrastés, avec une alternative textuelle aux images, et peut être navigué uniquement avec le clavier.

Référencement et SEO : (Search Engine Optimization) ou référencement naturel, désigne l’ensemble des techniques et bonnes pratiques utilisées pour optimiser la visibilité d’un site web dans les résultats des moteurs de recherch
	•	optimisation des contenus (mots-clés, titres, balises HTML, hiérarchie des pages)

Domaines de compétences

Le développement front-end englobe les langages suivants. 
	•	HTML 
	•	CSS & ses variantes LESS, SASS, COMPASS
	•	Javascript ses outils jQuery… ses variantes Angular, React… ou encore ses versions précompilées Typescript ou CoffeeScript

Structure d’un site web 

Pour inspecter la structure d’une page web, ouvrez l'inspecteur d'éléments de votre navigateur. 

Sur Windows, faites  Ctrl+Maj+C  . Sur macOS, faites  Option+Cmd+C  . 

Le code que vous voyez dans l'inspecteur est du code HTML – le langage qui définit la structure d'une page web.


Une page web est composée d'éléments – du texte, les images, les boutons…ce sont tous des exemples d'éléments. Un élément peut être parent de plusieurs éléments enfants.


Découvrez l’inspecteur de styles

L'inspecteur des navigateurs nous permet aussi de visualiser et de modifier le code qui régit les styles d'une page web, c'est-à-dire l'apparence des différents éléments. 
Dans l'inspecteur des styles ("Styles" dans Chrome, "Règles" ou "Rules" dans Firefox), vous pouvez filtrer les styles pour trouver ce qui vous intéresse, et même en modifier la valeur.


Le fait de pouvoir modifier les styles d'une page web nous permet de tester des modifications sans toucher au code source de notre projet.

Étudiez la console

La console nous apporte des informations essentielles sur le fonctionnement de nos projets, surtout en cas de problème. Elle vous sera notamment très utile lors de vos développement en Javascript.


Activez le mode mobile :

Environ 60 % du trafic web aujourd'hui se déroule sur mobile, donc il vous est indispensable, en tant que développeur front-end, de vérifier l'apparence de vos projets sur différents types d'appareils.


En mode mobile, l'en-tête du navigateur permet de choisir le type de mobile simulé, son orientation, et vous pouvez même choisir de limiter la bande passante pour simuler une mauvaise connexion.
